# Form Patterns

Arkite UI provides **layout-only** form components. They handle structure, labels, error display,
and accessibility — but **do not own form state or validation**.

This design lets you use any form library: `react-hook-form`, `formik`, native `useState`, or none at all.

## Components

| Component | Purpose |
|-----------|---------|
| `Form` | `<form>` wrapper with `disabled` context |
| `FormField` | Single field container with `name`, `error`, context |
| `FormLabel` | Connected `<label>` with `required`/`optional` indicator |
| `FormControl` | Wrapper for the actual input element |
| `FormDescription` | Help text below the input |
| `FormMessage` | Error message (reads from `FormField` context) |
| `FormSection` | `<fieldset>` with title and description |
| `FormActions` | Button row with alignment options |

## Basic Usage

```tsx
import { Form, FormField, FormLabel, FormControl, FormMessage, FormActions } from '@arkite/ui'
import { Input, Button } from '@arkite/ui'

<Form onSubmit={handleSubmit}>
  <FormField name="email" error={errors.email}>
    <FormLabel required>Email</FormLabel>
    <FormControl>
      <Input type="email" error={!!errors.email} />
    </FormControl>
    <FormMessage />
  </FormField>

  <FormActions>
    <Button type="submit">Save</Button>
  </FormActions>
</Form>
```

## With react-hook-form + zod

```tsx
import { useForm } from 'react-hook-form'
import { zodResolver } from '@hookform/resolvers/zod'
import { z } from 'zod'
import { Form, FormField, FormLabel, FormControl, FormMessage, FormActions } from '@arkite/ui'
import { Input, Button } from '@arkite/ui'

const schema = z.object({
  name: z.string().min(1, 'Name is required'),
  email: z.string().email('Invalid email'),
})

type FormValues = z.infer<typeof schema>

function MyForm() {
  const { register, handleSubmit, formState: { errors, isSubmitting } } = useForm<FormValues>({
    resolver: zodResolver(schema),
  })

  const onSubmit = async (data: FormValues) => {
    await api.save(data)
  }

  return (
    <Form onSubmit={handleSubmit(onSubmit)}>
      <FormField name="name" error={errors.name?.message}>
        <FormLabel required>Name</FormLabel>
        <FormControl>
          <Input {...register('name')} error={!!errors.name} />
        </FormControl>
        <FormMessage />
      </FormField>

      <FormField name="email" error={errors.email?.message}>
        <FormLabel required>Email</FormLabel>
        <FormControl>
          <Input {...register('email')} error={!!errors.email} />
        </FormControl>
        <FormMessage />
      </FormField>

      <FormActions>
        <Button type="submit" loading={isSubmitting}>Save</Button>
      </FormActions>
    </Form>
  )
}
```

## Dynamic Form Pattern

When your form fields come from a database or API (e.g. custom fields, CRM field definitions),
use `Form` + `FormField` with a field config array. **Do not build a separate `DynamicForm` component** —
compose with existing primitives.

```tsx
import { Form, FormField, FormLabel, FormControl, FormSection } from '@arkite/ui'
import { Input, Select, Textarea } from '@arkite/ui'

// Field definition from your API / database
interface FieldDefinition {
  key: string
  label: string
  type: 'text' | 'email' | 'number' | 'date' | 'select' | 'textarea'
  required?: boolean
  options?: { value: string; label: string }[]  // for select type
}

// Generic renderer — put this in your project's shared/ folder
function DynamicFields({
  fields,
  data,
  onChange,
  disabled,
}: {
  fields: FieldDefinition[]
  data: Record<string, any>
  onChange: (key: string, value: any) => void
  disabled?: boolean
}) {
  return (
    <div className="grid grid-cols-1 md:grid-cols-2 gap-4">
      {fields.map((field) => (
        <FormField key={field.key} name={field.key}>
          <FormLabel required={field.required}>{field.label}</FormLabel>
          <FormControl>
            {field.type === 'select' ? (
              <Select
                options={field.options ?? []}
                value={data[field.key] ?? ''}
                onChange={(e) => onChange(field.key, e.target.value)}
                disabled={disabled}
                placeholder={`Select ${field.label}...`}
              />
            ) : field.type === 'textarea' ? (
              <Textarea
                value={data[field.key] ?? ''}
                onChange={(e) => onChange(field.key, e.target.value)}
                disabled={disabled}
              />
            ) : (
              <Input
                type={field.type}
                value={data[field.key] ?? ''}
                onChange={(e) => onChange(field.key, e.target.value)}
                disabled={disabled}
              />
            )}
          </FormControl>
        </FormField>
      ))}
    </div>
  )
}
```

Usage:

```tsx
<Form onSubmit={handleSubmit}>
  <FormSection title="Basic Info">
    {/* Static fields */}
    <FormField name="name">...</FormField>
  </FormSection>

  <FormSection title="Custom Fields">
    {/* Dynamic fields from API */}
    <DynamicFields
      fields={fieldDefinitions}
      data={formData}
      onChange={(key, value) => setFormData(prev => ({ ...prev, [key]: value }))}
    />
  </FormSection>

  <FormActions>
    <Button type="submit">Save</Button>
  </FormActions>
</Form>
```

Key points:
- Use `Select` from `@arkite/ui` instead of native `<select>` — it supports `options` array, placeholder, error state, and theming.
- The renderer lives in **your project**, not in `@arkite/ui`.
- Extend field types as needed (e.g. `multi_select` → `Combobox`, `color` → `ColorPicker`).

## Key Principles

1. **Arkite UI owns layout, you own state.** The library never calls `useState` or manages form values.
2. **Pass `error` to `FormField`**, not to the input. `FormMessage` reads it from context automatically.
3. **Pass `error` (boolean) to `Input`** for red border styling. This is separate from the message.
4. **Use `FormSection`** for multi-step or grouped forms — it renders a semantic `<fieldset>`.
5. **Use `FormActions align="between"`** for split layouts (e.g. Reset on left, Save on right).
